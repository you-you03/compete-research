# Design Brief: 競合リサーチ ダッシュボード UI リデザイン

## 1. 背景・目的

車買取一括査定サービスのマーケティング担当者が使う**社内向け競合リサーチツール**のダッシュボードUIをリデザインしたい。

AIエージェント（Claude Code）が各競合企業をWebリサーチし、その結果を構造化JSONとして保存する。このJSONをもとに、1社ずつ深く読み込めるダッシュボードページをHTMLで生成する仕組みになっている。

現在の実装は情報密度は高いが、視覚的なメリハリがなく情報が並列で読みにくい。**Bento UIとポップなカラーリング**で、情報の重み・優先度が一目でわかるリデザインを依頼したい。

---

## 2. 成果物の仕様

| 項目 | 内容 |
|------|------|
| **形式** | 単一HTMLファイル（`dashboard.html`）にCSSとJSをインライン |
| **フレームワーク** | 不要。Pure HTML / CSS / Vanilla JS のみ |
| **データ渡し方** | HTMLファイル内の `<script>` タグに `const COMPANIES_DATA = [...]` としてJSONをハードコード。後述のサンプルJSONをそのまま使ってよい |
| **ターゲット環境** | デスクトップ（MacのChromeで `open dashboard.html` で開く）。モバイル対応は不要 |
| **生成タイミング** | このHTMLはPythonスクリプトが動的に生成する。デザインだけ作ってもらえれば、Python側への組み込みは自分でやる |

---

## 3. ナビゲーション設計

### 会社切り替えUI

- 画面左側に**縦サイドバー**を配置し、調査済み企業の一覧をリスト表示
- 企業名をクリックすると右側のメインエリアのBentoグリッドが切り替わる
- アクティブな企業はハイライト表示
- 企業名の横に**優先度バッジ**（high / medium / low / self）を表示

```
┌──────────────┬─────────────────────────────────────────┐
│  サイドバー  │  メインエリア（Bento Grid）               │
│              │                                         │
│ ▶ MOTA [自社] │  [選択中の会社のBentoグリッド]           │
│   ナビクル   │                                         │
│   マイカース │                                         │
│   ...        │                                         │
└──────────────┴─────────────────────────────────────────┘
```

## 4. Bentoグリッドのレイアウト設計

メインエリアは**6カラムグリッド**で構成する。各タイルはカラムスパンで大きさを指定。行の高さは内容に合わせて可変。タイル間のgapは `16px`。

### タイル構成と優先度

```
Row 1: [──────────────── 6col ────────────────]
        HERO: 会社名 + LP キャッチコピー + サマリー

Row 2: [── 2col ──] [──────── 4col ────────]
        規模感KEY    サービスフロー（ステップ）

Row 3: [────── 3col ──────] [────── 3col ──────]
        ビジネスモデル        差別化要因・独自機能

Row 4: [── 2col ──] [────── 2col ──────] [── 2col ──]
        ユーザー評判   マーケティング      採用シグナル

Row 5: [──────────────── 6col ────────────────]
        最新動向（タイムライン）
```

### 各タイルの詳細仕様

#### 🟫 Row 1: HERO タイル（6col、大）
- 背景: ダークネイビー `#0f172a`、テキスト白
- 左側: 会社名（大見出し）+ 公式URLリンク + 優先度バッジ
- 中央: LP キャッチコピーを大きなイタリック体で引用表示（`"買取店が競うから高くなる"`のように）
- 右側: サマリー文章（3〜5文）
- データキー: `company_id`, `details.differentiation.lp_catchcopy`, `summary`

#### 🟡 Row 2 左: 規模感タイル（2col）
- 背景: アンバー `#f59e0b`、テキスト白
- **数字を超大きく表示**。以下の情報をカード内に並べる
  - 月間利用者数（例: `78,000人/月`）
  - 提携業者数（例: `1,500社+`）
  - 成長率（例: `CAGR 205%`）
- 数字が取得できていない場合は「—」で表示（グレーアウト）
- データキー: `details.scale.users`, `details.service.partners`, `details.scale.assessments`

#### 🔵 Row 2 右: サービスフロータイル（4col）
- 背景: スカイブルー `#0ea5e9`、テキスト白
- **ステップを横並びに表示**（→ で連結するフロー図風）
- ステップ数が多い場合は2行に折り返してよい
- 各ステップは番号付き丸アイコン + テキスト
- フロー下部に: 対応エリア、最短査定時間、提携業者を小さく補足
- データキー: `details.service.flow_steps`, `details.service.speed`, `details.service.coverage_area`

#### 🔴 Row 3 左: ビジネスモデルタイル（3col）
- 背景: コーラルレッド `#f87171`、テキスト白
- セクションタイトル「💰 ビジネスモデル」
- 以下をリスト表示:
  - 収益タイプ（例: 成約手数料型）
  - ユーザー費用
  - 業者費用構造
  - 情報開示タイミング
  - 副収益（あれば）
- データキー: `details.business_model.*`

#### 🟢 Row 3 右: 差別化要因タイル（3col）
- 背景: エメラルドグリーン `#10b981`、テキスト白
- セクションタイトル「⭐ 差別化要因」
- 独自機能をリストで表示（特許・独占機能は太字 or 🏆 マーク）
- 下部に: ターゲットユーザー、権威付け実績（No.1等）をタグ/バッジで表示
- データキー: `details.differentiation.unique_features`, `details.differentiation.target_users`, `details.differentiation.authority_claims`

#### 🩷 Row 4 左: ユーザー評判タイル（2col）
- 背景: ライトクリーム `#fff7ed`、テキスト `#1a1a1a`
- 上段: 👍 好評ポイント（緑のドット付きリスト）
- 下段: 👎 不満ポイント（赤のドット付きリスト）
- データキー: `details.reputation.positive_themes`, `details.reputation.negative_themes`

#### 🟣 Row 4 中: マーケティングタイル（2col）
- 背景: バイオレット `#7c3aed`、テキスト白
- TVCM情報（タレント名・放映時期）を目立たせる
- SEO状況、SNSフォロワー、広告テーマを続けて表示
- データキー: `details.marketing.*`

#### 🟩 Row 4 右: 採用シグナルタイル（2col）
- 背景: ライムグリーン `#84cc16`、テキスト `#1a1a1a`
- セクションタイトル「🏢 採用シグナル（投資領域の手がかり）」
- 募集ポジションを箇条書きリスト
- データキー: `details.hiring_signals`

#### 🌑 Row 5: 最新動向タイル（6col）
- 背景: ダークスレート `#1e293b`、テキスト白
- **横スクロール可能なタイムライン**形式
- 各ニュースアイテム: 日付（小文字） + タイトル（太字） + 概要テキスト
- カード風に横並び（overflow-x: auto）
- データキー: `details.news[]` （`{date, title, summary}` の配列）

## 6. タイポグラフィ・スタイルの方向性

- フォント: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Hiragino Sans, sans-serif`
- タイル角丸: `border-radius: 20px`（大きめにして柔らかく）
- タイル間ギャップ: `16px`
- 各タイルにはわずかな `box-shadow` を付ける: `0 2px 12px rgba(0,0,0,0.08)`
- 数字（規模感タイルなど）は `font-size: 2.5rem; font-weight: 800` で大きく
- セクション見出しは絵文字付きで視認性アップ
- テキストが白背景のタイルではアイコン/バッジで視覚アクセントを足す

## 7. データ構造（JSONスキーマ）

HTMLファイルには以下の形式でデータをインライン埋め込みすること。
`companies_meta` は `data/companies.json` の情報（URL・優先度など）。

```javascript
const COMPANIES_DATA = [
  {
    // companies.json のメタ情報
    "id": "mota",
    "name": "MOTA（株式会社MOTA）",
    "url": "https://mota.jp",
    "priority": "self",

    // 以下は research cache の内容（下記サンプルJSON参照）
    "summary": "...",
    "details": {
      "service": { ... },
      "business_model": { ... },
      "differentiation": { ... },
      "scale": { ... },
      "company": { ... },
      "marketing": { ... },
      "reputation": { ... },
      "news": [ { "date": "YYYY-MM", "title": "...", "summary": "..." } ],
      "hiring_signals": [ "..." ]
    },
    "data_gaps": [ "..." ],
    "collected_at": "2026-04-25T12:00:00"
  }
];
```

## 9. 表示しなくてよいもの

以下のデータは今回のUIには**表示不要**（データは存在するが視覚化しない）:

- `scores`（脅威度・サービス完成度・マーケ強度のスコア数値）
- `data_gaps`（取得できなかった情報のリスト）
- `details.company.revenue`, `details.company.funding`（財務情報は今回省略）

## 10. 実装時の注意点

1. **データがない場合の表現**: フィールドが `null`、空文字、`"—"` の場合はそのタイルの該当行をグレーの `—` で表示するか、行自体を省略する
2. **テキストの長さ**: 各タイルは固定高にしないこと。内容に応じて縦に伸びてよい。ただし `flow_steps` が多い場合は2列表示にしてよい
3. **ニュースタイムライン**: アイテム数が多い場合（7件以上）は横スクロールで対応
